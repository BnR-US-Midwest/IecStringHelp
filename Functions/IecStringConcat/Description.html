Description

In an IEC function declaration, a parameter of type STRING[5] is equivalent to a parameter of type STRING[10]. STRINGs are passed as pointers (to the array of characters) in library functions.

Therefore, the STRING parameters are denoted as STRING[0] because the size is not checked. The size must be explicitely passed as an additional function parameter.

ADR() is not necessary for the string parameters of this function because this function is expecting a STRING[] (pointer to) and not a UDINT.

These details can be observed in the automatically generated library header, specifically the function prototypes.

    _BUR_PUBLIC signed long IecStringConcat(plcstring* destination, unsigned long size, plcstring* source);

Examples

IEC Structured Text

    VAR
        String1 : STRING[8];
        String2 : STRING[10] := '56789';
    END_VAR
  
    PROGRAM _CYCLIC
        IecStringCopy(String1, SIZEOF(String1), '01234');
        IecStringConcat(String1, SIZEOF(String1), String2);
    END_PROGRAM

The value of String1 is '0123456'.
